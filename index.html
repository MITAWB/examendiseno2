Tema 4 · Modelo de cajas + Frameworks (YUI / Bootstrap 4)

CSS box model + idea de frameworks + container/grid/colores/tipografía

Tema 4
Modelo de cajasPregunta típica

En CSS, todo elemento se representa como una caja. Sus capas:

  • Content: el contenido (texto/imagen). Se controla con width/height.
  • Padding: relleno interno entre content y borde.
  • Border: borde alrededor del padding/content.
  • Margin: espacio externo que separa cajas.
Lo que cae: saber qué es cada capa, y que el tamaño visual total = content + padding + border (+ margen como separación).
YUI (Yahoo! User Interface)

Framework (histórico) de utilidades y buenas prácticas en CSS y JavaScript. Incluía ficheros típicos como reset/fonts/grids para normalizar estilos y definir rejillas.

  • Idea: acelerar diseño con una base consistente.
  • Enlace típico: reset-fonts-grids.css.
Bootstrap 4Framework moderno

Framework para maquetar rápido y responsive. Lo clave:

  • Container: .container (con márgenes) vs .container-fluid (ancho completo).
  • Grid: filas .row y columnas .col-* (sistema 12 columnas).
  • Tipografías y colores por clases: text-*, bg-*, etc.
Mini ejemploLo que podrías hacer en práctica
<div class="container">
  <div class="row">
    <div class="col-12 col-md-6">Mitad en md+</div>
    <div class="col-12 col-md-6">Mitad en md+</div>
  </div>
</div>

Interpretación: móvil (col-12) → una debajo de otra. En md+ (col-md-6) → dos columnas.

Tipo test rápido (Tema 4): ¿Qué capa del box-model separa una caja de otra?
Margin. El padding es interno; el margin es espacio externo.
Tipo test rápido (Tema 4): Diferencia entre container y container-fluid
container mantiene márgenes laterales; container-fluid ocupa todo el ancho.
Box-model + frameworks: base para todo DIW

Tema 5 · Preprocesadores (SASS / LESS)

Variables, mixins, operadores, anidamiento y estructura de ficheros

Tema 5
Definición

Un preprocesador te deja escribir “CSS mejorado” (variables, mixins, nesting, operadores) y luego se compila a CSS normal.

Idea de examen: antes, si querías cambiar un color global, tocabas muchas líneas; con variables, cambias 1 valor y recompilas.
Estructura recomendada (SCSS)
  • Separar ficheros por responsabilidad: config, utilities, mixins, componentes, layout…
  • Usar @import (o la forma moderna) para componer un CSS final.
// main.scss
@import "config";
@import "utilities";
@import "mixins";
VariablesSASS / LESS

Guardas valores (colores, tamaños, etc.) para reutilizarlos.

// SASS
$primary: #0d6efd;
.btn { background: $primary; }
// LESS
@primary: #0d6efd;
.btn { background: @primary; }
MixinsReutilización

Bloques reutilizables, como “funciones” de CSS.

// SASS
@mixin pill($bg){
  padding: 8px 12px;
  border-radius: 999px;
  background: $bg;
}
.tag { @include pill(rgba(110,231,255,.12)); }
Operadores + anidamiento

Puedes hacer cálculos y anidar selectores para organizar jerarquías. 

// SASS
$gap: 12px;

.card{
  padding: $gap;
  h3{ margin-top: $gap * 1.5; }
}

En CSS final se “aplana” a selectores normales.

Identación + Grid system (idea)

En el tema se menciona la identificación/estructura jerárquica en SASS y el concepto de “grid system” como sistema de rejilla/columnas, muy útil para layouts consistentes.

Tipo test rápido: ¿Qué es un mixin?
Un bloque de código reutilizable (tipo función) que puede recibir parámetros y se inserta donde lo incluyes.
Tipo test rápido: ¿Cuál es la ventaja de variables?
Cambias un valor en un sitio y afecta a todos los usos al recompilar.
Memoriza: variables · mixin · nesting · operadores · compilar → CSS

Tema 9 · Integración de contenido interactivo (jQuery)

DOM, ready, funciones DOM, eventos (teclado/ventana/ratón), css() y replaceWith()

Tema 9
Qué es

jQuery es una librería de JavaScript que facilita la interacción con el DOM: seleccionar elementos, modificar su contenido/estilos y escuchar eventos.

Cae muchísimo: DOM + ready + selectores + eventos + css() + replaceWith().
Ready (DOM listo)
$(document).ready(function(){
  // tu código aquí
});

// versión corta:
$(function(){ /* ... */ });

Se usa para asegurar que el DOM existe antes de manipularlo.

Selectores
  • $("#id") por id
  • $(".clase") por clase
  • $("div") por etiqueta
  • $("input[name=email]") por atributo
// encadenado típico
$("#msg").addClass("ok").text("Guardado");
Funciones DOM
  • .text() texto
  • .html() HTML
  • .val() value inputs
  • .append() / .prepend() insertar
css()Tema

Modifica estilos desde JS.

$("#caja").css("background", "#111827");
$("#caja").css({ color:"#fff", padding:"12px" });
replaceWith()Tema

Reemplaza un elemento por otro nuevo.

$("#banner").replaceWith("<div id='banner'>Nuevo</div>");
Eventos por categorías
Teclado
  • keydown
  • keyup
  • keypress (menos usado)
Ventana
  • resize
  • scroll
  • load
Ratón
  • click
  • mouseenter / mouseleave
  • mousemove
$(document).on("keydown", function(e){
  if(e.key === "Escape"){ $("#modal").hide(); }
});
Tipo test rápido: ¿Qué hace replaceWith()?
Sustituye el elemento seleccionado por un nuevo elemento/HTML.
Tipo test rápido: ¿Qué diferencia hay entre text() y html()?
text() escribe texto plano; html() interpreta etiquetas HTML.
Recuerda: ready → select → evento → DOM/css/replace

Tema 10 · Accesibilidad Web (A11y)

Conceptos, niveles de adecuación, herramientas (Contrast/WAVE/TAW) y CSS auditivo

Tema 10
Definición

Accesibilidad = diseñar para que cualquier usuario pueda acceder y utilizar la web con facilidad, reduciendo barreras (visuales, auditivas, motrices, cognitivas…).

En examen: niveles, herramientas y checklist (alt, contraste, teclado, semántica).
Principios + niveles
  • Principios habituales: perceptible, operable, comprensible, robusto.
  • Niveles de adecuación: A (mínimo), AA (recomendado), AAA (máximo).
Checklist rápido
  • Imágenes con alt útil
  • Contraste texto/fondo suficiente
  • Navegación por teclado (Tab/Enter) y foco visible
  • Labels en formularios + errores claros
  • Estructura semántica (h1-h2…, nav, main…)
Herramienta: Color Contrast Checker

Comprueba si el contraste entre texto y fondo cumple pautas de accesibilidad.

En el tema se menciona dentro de WebAIM.

Herramienta: WAVE

Web Accessibility Evaluation Tool: analiza accesibilidad y muestra errores/avisos de forma visual sobre la página.

Herramienta: TAW

Familia de herramientas para medir el grado de accesibilidad. Incluye análisis y certificaciones según el resultado.

Hojas de estilo auditivas (CSS Speech Module)

Estilos pensados para reproducir/interpretar contenido por voz (útiles para usuarios con dificultad visual). En el tema se tratan como “hojas de estilo auditivas” y sus propiedades.

Si te lo preguntan en desarrollo: explica qué son y para qué sirven (adaptación a salida por audio).
Tipo test rápido: Orden correcto de niveles
A → AA → AAA (de menos a más exigente).
Tipo test rápido: ¿Para qué sirve WAVE?
Para evaluar accesibilidad y ver problemas/avisos de forma visual sobre la web.

Tema 11 · Usabilidad Web

Concepto, tipos de usuario, barreras, navegación, normativa ISO y técnicas de evaluación

Tema 11
Definición

Usabilidad = facilidad con la que un usuario completa tareas en una web de forma eficiente, aprendible y con buena experiencia.

Diferencia clave: Accesibilidad = “que todos puedan usarla”. Usabilidad = “que sea fácil y eficiente”.
Tipos de usuario
  • Novato: necesita guía, claridad y consistencia.
  • Intermedio: combina guía y atajos.
  • Experto: busca rapidez, atajos y control.
Barreras típicas
  • Navegación confusa (no sabes dónde estás)
  • Falta de feedback (no confirma acciones)
  • Formularios largos o con errores poco claros
  • Demasiada carga visual / exceso de elementos
Navegación en la web
  • Menús claros + jerarquía coherente
  • Enlaces con texto significativo
  • “Miguitas” / indicación de ubicación
  • Consistencia entre páginas
Normativa ISO (del tema)
  • ISO/IEC 9241: guía de usabilidad y medidas evaluadas.
  • ISO 13407: diseño de sistemas interactivos centrado en el usuario.
  • ISO/TR 9241/151: directrices para diseño de interfaces web centrado en el usuario.
Técnicas de análisis y verificación (idea de examen)
  • Pruebas con usuarios (observación de tareas)
  • Heurísticas (revisión experta)
  • Métricas: tiempo, errores, satisfacción
  • Herramientas para evaluar navegación y problemas

En desarrollo suelen pedir “cómo verificarías la usabilidad” + ejemplos.

Tipo test rápido: ¿Qué ISO se asocia a “diseño centrado en el usuario” en sistemas interactivos?
ISO 13407.
Usabilidad: usuarios + barreras + navegación + ISO + verificación

Tema 12 · Bibliotecas de interfaces (I) — MVC + React

Patrones (MVC), herramientas (Node/npm/npx), componentes, props, state, render y composición

Tema 12
Patrón MVC

MVC divide una app en 3 partes:

  • Modelo: datos y lógica de negocio
  • Vista: interfaz (lo que ve el usuario)
  • Controlador: conecta vista y modelo (gestiona acciones)
En el tema se comenta que el desarrollo JS moderno suele apoyarse en patrones como MVC.
Herramientas React
  • Node.js: entorno para ejecutar herramientas JS
  • npm: gestor de paquetes
  • npx: ejecutar paquetes/comandos sin instalar global
Componentes

Un componente es una pieza reusable con 3 pilares:

  • Estructura (JSX)
  • Estilos
  • Comportamiento
Props

Objeto de entrada: valores que recibe el componente para hacerlo dinámico.

const Saludar = (props) => (
  <h3>Hola {props.name}</h3>
);
State

Estado interno (cambia con interacción).

// clase (según temario)
this.state = { contador: 0 };
this.setState({ contador: this.state.contador + 1 });
Render + composición

Renderizar = pintar el componente en el DOM. La composición = un componente usa otros dentro. 

function App(){
  return (
    <div>
      <Saludar name="Diana" />
      <Saludar name="Pepe" />
    </div>
  );
}

ReactDOM.render(<App />, document.getElementById("root"));

Eventos en React

Routing

Permite navegación por rutas (home/tienda/galería) en SPA. React Router se usa en web con react-router-dom.

Optimización de rutas

En vez de escribir muchas rutas a mano, defines un array de rutas y lo mapeas. 

const routes = [
  { path: "/", component: Home },
  { path: "/tienda", component: Tienda },
  { path: "/galeria", component: Galeria },

Preguntas de desarrollo (con respuesta modelo)

Desarrollo
Desarrollo 1

Explica el modelo de cajas de CSS y pon un ejemplo de cómo afecta al tamaño final de un elemento.

El modelo de cajas representa cada elemento como una caja formada por content, padding, border y margin. El content es el área del contenido (controlada por width/height), el padding es el relleno interno, el border rodea al content+padding y el margin es el espacio externo que separa cajas. El tamaño visible total del elemento incluye content + padding + border; el margin no suma al “tamaño del elemento” pero sí ocupa espacio alrededor para separarlo.

Desarrollo 2

Describe Bootstrap 4: container, grid y cómo lo usarías para un diseño responsive.

Bootstrap 4 es un framework CSS/JS orientado a diseño responsive. Se usa un contenedor .container o .container-fluid. Dentro se crean filas con .row y columnas con .col-* en un sistema de 12 columnas. Con breakpoints (por ejemplo col-12 col-md-4) se adapta el layout: móvil a una columna, escritorio a varias. Además incluye clases de tipografía, colores y componentes listos (cards, navbar, formularios).

Desarrollo 3

Explica qué es accesibilidad web y cómo la evaluarías con herramientas del tema.

La accesibilidad busca que cualquier usuario pueda usar la web reduciendo barreras (visuales, motrices, cognitivas…). Se revisan aspectos como texto alternativo, contraste, navegación por teclado, semántica y mensajes de error. Para evaluar: Color Contrast Checker para contraste, WAVE para un análisis visual de errores/avisos, y TAW para medir el grado de accesibilidad y obtener análisis/certificaciones. También se contemplan hojas de estilo auditivas para adaptar contenido a salida por audio (CSS Speech).

Desarrollo 4

Explica React con MVC y termina describiendo routing + Flux (idea general).

MVC separa una app en Modelo (datos), Vista (interfaz) y Controlador (conecta y gestiona acciones). React se centra en la Vista, construyendo UI con componentes (estructura en JSX, estilos y comportamiento). Los componentes reciben props (entrada) y gestionan state (estado interno). Para navegación en SPA se usa React Router, que asigna rutas (URLs) a componentes. Flux es una arquitectura de flujo unidireccional que separa Views, Actions, Dispatcher y Stores, haciendo más controlable el estado y el flujo de datos.

Truco: en desarrollo, estructura: definición → partes → ejemplo

<form id="formUsuario">
  <label>Nombre</label><br>
  <input id="nombre"><br><br>

  <label>Email</label><br>
  <input id="email"><br><br>

  <button type="submit">Añadir</button>
  <button type="button" id="btnRemove">Borrar último</button>

  <div id="msg"></div>
</form>

<ul id="lista"></ul>

<script src="https://cdn.jsdelivr.net/npm/jquery@3.7.1/dist/jquery.min.js"></script>
<script>
$(function () {

  function setMsg(texto, ok){
    $("#msg").css("color", ok ? "green" : "crimson").text(texto);
  }

  $("#formUsuario").on("submit", function(e){
    e.preventDefault();

    const nombre = $("#nombre").val().trim();
    const email = $("#email").val().trim();

    if(!nombre) return setMsg("Nombre obligatorio", false);
    if(!email) return setMsg("Email obligatorio", false);
    if(!email.includes("@")) return setMsg("Email debe contener @", false);

    $("#lista").append("<li><b>" + nombre + "</b> — " + email + "</li>");
    $("#nombre").val("");
    $("#email").val("");
    setMsg("Añadido", true);
  });

  $("#btnRemove").on("click", function(){
    const items = $("#lista li");
    if(items.length === 0) return setMsg("Nada que borrar", false);
    items.last().remove();
    setMsg("Borrado", true);
  });

});
</script>
